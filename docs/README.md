# Why This Project Exists

Kasir App dibuat bukan karena dunia kekurangan aplikasi POS.

Justru sebaliknya.

Sudah terlalu banyak aplikasi kasir yang:
- penuh fitur
- terlihat modern
- memiliki dashboard kompleks
- membutuhkan perangkat mahal
- tetapi akhirnya jarang benar-benar dipakai dalam operasional harian.

Project ini lahir dari pengalaman melihat langsung banyak usaha kecil dan retail tradisional yang sebenarnya sudah memiliki mesin kasir atau software POS, namun pada akhirnya kembali menggunakan:
- kalkulator
- catatan manual
- ingatan pribadi
- pencatatan sederhana

Bukan karena mereka menolak teknologi.

Tetapi karena banyak software POS modern terasa:
- terlalu rumit
- terlalu corporate-oriented
- terlalu banyak langkah
- terlalu sensitif terhadap kesalahan kecil
- dan justru menambah beban operasional.

Dalam beberapa kasus, kesalahan sederhana seperti salah input tanggal atau salah klik dapat menyebabkan:
- transaksi error
- laporan tidak sinkron
- stok menjadi kacau
- hingga pengguna takut menggunakan sistemnya sendiri.

Dari situlah muncul pertanyaan sederhana:

> apakah software benar-benar membantu operasional,
> atau justru membuat pekerjaan menjadi lebih melelahkan?

Kasir App dibangun dengan filosofi berbeda.

Tujuan project ini bukan membuat sistem POS paling mewah atau paling kompleks.

Tujuan utamanya adalah:
- membantu operasional harian menjadi lebih ringan
- menjaga workflow tetap sederhana
- mengurangi friksi penggunaan
- tetap nyaman digunakan di perangkat Android sederhana
- dan tetap stabil untuk usaha nyata sehari-hari

Project ini lebih memprioritaskan:
- stabilitas transaksi
- kecepatan penggunaan
- inventory yang mudah dipahami
- ketahanan terhadap kesalahan pengguna
- dan pengalaman kasir yang praktis

dibanding sekadar:
- dashboard berlebihan
- fitur enterprise terlalu dini
- animasi kompleks
- atau kompleksitas yang tidak benar-benar dibutuhkan UMKM.

Kasir App percaya bahwa software yang baik bukan hanya software yang terlihat modern,
tetapi software yang benar-benar dipakai dan membantu pekerjaan sehari-hari.

# Why This Project Exists

Kasir App was not created because the world lacks POS applications.

In fact, there are already many POS systems that are:
- feature-rich
- visually modern
- packed with advanced dashboards
- designed for expensive hardware
- yet rarely used comfortably in real daily operations.

This project was born from direct observations of small businesses and traditional retail stores that already owned POS devices or cashier software, but eventually returned to:
- calculators
- handwritten notes
- personal memory
- simple manual bookkeeping

Not because they rejected technology.

But because many POS systems felt:
- too complicated
- too corporate-oriented
- too rigid
- too sensitive to small mistakes
- and ultimately added more operational stress instead of reducing it.

In some situations, simple mistakes such as entering the wrong date or clicking the wrong button could cause:
- transaction failures
- broken reports
- inconsistent inventory
- or even make users afraid to use the system itself.

That led to a simple question:

> Is the software truly helping the business,
> or is it making daily work even harder?

Kasir App is built with a different philosophy.

The goal of this project is not to create the most advanced or most feature-complete POS system.

The primary goal is to:
- reduce operational friction
- keep workflows simple
- make daily transactions easier
- remain lightweight on low-end Android devices
- and stay reliable for real-world daily business operations

This project prioritizes:
- transaction stability
- usability
- lightweight inventory management
- resilience against user mistakes
- and practical cashier workflows

instead of focusing too early on:
- excessive dashboards
- enterprise-level complexity
- unnecessary animations
- or features that small businesses do not truly need.

Kasir App believes that good software is not software that only looks modern,
but software that is genuinely useful and comfortable to use every single day.
